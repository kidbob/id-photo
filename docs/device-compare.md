# 真机对比清单：快捷指令 vs PWA

在同一台 iPhone、同一张原图上对比，填写下表便于决定是否把抠图也迁入 PWA。

## 测试准备

- [ ] 原图：同一张半身证件照（建议 2000px 以上宽度）
- [ ] 快捷指令已按 [shortcut-setup.md](./shortcut-setup.md) 配置
- [ ] PWA 已添加到主屏幕（`npm run dev` 同 WiFi 或 `dist` 部署 HTTPS）
- [ ] 测试规格：**一寸 295×413**，上传上限 **50 KB**

## 推荐结论（测试后勾选）

- [ ] **维持组合方案**：快捷指令抠图 + PWA 压 KB（计划默认推荐）
- [ ] **仅快捷指令**：接受手调 JPEG 或不要 50KB 限制
- [ ] **PWA 单应用**：愿接受浏览器 WASM 抠图（需另加依赖，边缘可能略逊）

## 典型预期（供对照）

| 场景 | 预期 |
|------|------|
| 快捷指令抠图 | 10–30 秒；边缘通常优于浏览器 WASM |
| PWA 仅压缩 | 1–3 秒；50KB 下一寸照 quality 约 40–70% |
| 组合总流程 | 先快捷指令出 PNG，再 PWA 压 JPEG，清晰度优于单快捷指令压 KB |

## 问题排查

| 现象 | 处理 |
|------|------|
| PWA 打不开 | 检查 WiFi/HTTPS；Safari 非无痕 |
| 导出仍 &gt; 50KB | 换更小规格或接受当前最优压缩 |
| 透明底未铺色 | PWA 选底色后重新「处理并预览」 |
| 快捷指令无抠图动作 | 升级 iOS 至 16+ |
